# Dark Mode Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add system-preference-based dark mode using pure CSS custom property overrides.

**Architecture:** A `@media (prefers-color-scheme: dark)` block in `style.css` redefines all color variables. Hardcoded colors in styled-components are replaced with CSS variable references. No JavaScript required.

**Tech Stack:** CSS custom properties, `prefers-color-scheme` media query, styled-components (existing)

- [ ] **Step 1: Add `--color-surface` to `:root`**


- [ ] **Step 2: Add the dark mode media query**

After the `:root` closing brace (line 67), add:

```css
@media (prefers-color-scheme: dark) {
  :root {
    --color-primary: #6db3f2;
    --color-text: #e0e0e0;
    --color-text-light: #888888;
    --color-heading: #e0e0e0;
    --color-heading-black: #ffffff;
    --color-accent: #2a2a2a;
    --color-hover: #1a1a1a;
    --color-blockquote: #161616;
    --color-background: #000000;
    --color-surface: #1a1a1a;
  }
}
```

- [ ] **Step 3: Add dark Prism.js overrides**
